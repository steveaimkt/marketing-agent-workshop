# 4교시 실습 · 내 직군 에이전트 만들기

> 오늘의 최종 결과물이다. "대시보드(2교시) + 연결(3교시) + 내 규칙"을 하나로 묶으면,
> 명령 한 줄에 분석하고 화면·기록·요약까지 내보내는 에이전트가 된다.
> 트랙마다 만드는 에이전트가 다르다. 나머지는 전부 같다.

## 진행 원칙 (클로드가 지킬 것)

1. 한 단계씩 안내하고 수강생의 응답을 기다린다. 묶어서 빨리 넘기지 않는다.
2. 수강생이 입력할 문장은 반드시 코드블록으로 준다. 복붙만으로 진행되게 한다.
3. 실행 전에 "이런 결과가 나오면 성공"을 먼저 알려준다.
4. 3교시 연결에 실패한 사람도 그대로 완주시킨다. 실습 데이터로 하면 된다.
5. 진도가 느린 사람은 **필수 에이전트 1개만** 끝내면 성공이라고 안심시킨다.

## 진행 흐름 (5단계)

```
1단계  소개            에이전트가 뭔지, 무엇을 합치는지        ⏸ 답 대기
2단계  뭘 만들지 정하기  자동화 매트릭스 (막막한 사람용)        ⏸ 답 대기
3단계  합성            대시보드 + 연결 + 내 규칙을 하나로      ⏸ 답 대기
4단계  시나리오 적용     내 트랙 안에서 하나 골라 붙이기        ⏸ 답 대기
5단계  점검            정확한가 · 매번 같은가 · 안전한가       ⏸ 답 대기
```

---

## 1단계 · 소개

```
🤖 이제 오늘의 최종 결과물을 만듭니다.

🎯 에이전트가 뭔가요
  '명령 한 줄로 여러 단계를 한 번에 해주는 것'입니다. 명령 한 줄이면
    ① 데이터를 가져와  ② 계산하고  ③ 웹 화면으로 만들고
    ④ 노션에 저장하고  ⑤ 디스코드로 요약을 보냅니다.

🧩 무엇을 합치나요
  · 2교시 : 대시보드 만드는 방법을 저장해 둔 파일
  · 3교시 : 계정 연결 (실제 데이터 가져오기)
  · 여기에 '내 규칙'(며칠치를 볼지, 어떤 기준으로 경고할지)을 얹으면 = 에이전트

⏱ 이렇게 바뀝니다
  · 전 : 데이터 긁고, 표 만들고, 정리하고, 보고서 쓰고
  · 후 : 명령 한 줄이면 결과가 바로 나온다

📌 오늘 목표는 에이전트 1개입니다.
   빨리 끝나신 분은 하나 더 만듭니다. 1개만 완성해도 성공입니다.
```

⏸ 이해되셨으면 "2단계" 라고 답해 주세요.

## 2단계 · 무엇을 만들지 정하기 (자동화 매트릭스)

먼저 묻는다.

```
만들 것이 이미 정해진 분도 있고, 뭘 자동화해야 할지 막막한 분도 있습니다.

  A  내 트랙의 기본 에이전트로 바로 갈게요        → "A" 입력
  B  내 업무 중 뭘 자동화할지 모르겠어요           → "B" 입력
```

⏸ A 또는 B 응답 대기.

### A를 고른 경우
바로 3단계로 넘어간다. 트랙별 필수 에이전트는 3단계 표에 있다.

### B를 고른 경우 (자동화 매트릭스)

먼저 웹 진단 도구를 안내한다. 채점과 프롬프트 생성까지 자동으로 되기 때문에, 이 길이 훨씬 빠르다.

```
📊 자동화 매트릭스로 진단해 드릴게요. 두 가지 방법이 있습니다.

  ① 웹 도구로 하기 (추천 · 5분)
     https://likelion-marketing-automation-matrix.netlify.app

     · 내 직군을 고르면 예시 업무가 나옵니다. 막막하면 거기서 시작하세요.
     · 업무를 넣으면 우선순위 사분면에 자동으로 배치됩니다.
     · 스킬인지 에이전트인지 대시보드인지도 정해 줍니다.
     · 마지막에 나오는 프롬프트를 복사해 이 채팅창에 붙여넣으면 바로 구축으로 넘어갑니다.
     · 입력값은 각자 브라우저에만 저장됩니다. 서버로 보내지 않습니다.

  ② 여기 채팅에서 바로 하기
     반복하는 업무를 3~5개만 적어 주세요. 제가 채점해 드립니다.
     (예: "매주 월요일 광고 성과 취합해서 시트에 정리")

6번 1인 마케터 트랙이시면 데이터/6_1인마케터/tasks.md 에 예시 15개가 있습니다.
사전 안내 때 미리 진단해 오신 분은 그때 받은 프롬프트를 그대로 붙여넣으시면 됩니다.
```

⏸ 웹 도구에서 받은 프롬프트를 붙여넣거나, 업무 목록을 적어 주세요.

**프롬프트를 붙여넣은 경우** : 이미 진단이 끝난 상태다. 바로 3단계 합성으로 넘어간다.

**업무 목록을 적은 경우** : 각 업무를 5축으로 채점해 표로 보여 준다.

| 채점 축 | 무엇을 보나 | 점수 |
|:--|:--|:--|
| 빈도 | 얼마나 자주 하는가 | 1~5 |
| 소요 시간 | 한 번에 몇 분 걸리는가 | 1~5 |
| 반복성 | 매번 같은 순서로 하는가 | 1~5 |
| 데이터 명확성 | 입력 데이터가 정해져 있는가 | 1~5 |
| 판단 개입 | 사람 판단이 적게 필요한가 | 1~5 |

```
📊 진단 결과 (예시)

  업무                        빈도 시간 반복 데이터 판단  합계
  매주 광고 성과 취합            5    4    5    5     4    23  ← 1순위
  콘텐츠 초안 작성               4    5    2    2     2    15

  20점 이상이면 자동화 1순위입니다.
  1순위 업무를 그대로 오늘의 에이전트로 만들겠습니다.

  ⚠️ 점수가 낮게 나온 일은 자동화가 안 되는 게 아닙니다.
     '오늘 6시간 안에 만들기에 적합하지 않다'는 뜻입니다.
     판단이 많이 들어가는 일일수록 사람이 남아 있어야 합니다.
```

⏸ "이걸로 갈게요" 응답 대기.

## 3단계 · 합성 (대시보드 + 연결 + 내 규칙)

내 트랙에 해당하는 명령만 안내한다. 수강생은 복붙하면 된다.

### 트랙별 필수 에이전트

| 트랙 | 오늘 만들 에이전트 | 나오는 결과물 |
|:--|:--|:--|
| 1 퍼포먼스 | 광고 성과 분석 에이전트 | 광고 대시보드 + 기준 위반 경고 |
| 2 콘텐츠 | 콘텐츠 캘린더 에이전트 | 주간 기획안 + 초안 |
| 3 CRM | 고객 세그먼트 분석 에이전트 | 세그먼트 리포트 + 이탈 위험군 |
| 4 브랜드 | 브랜드 모니터링 에이전트 | 경쟁사 변경점 + VoC 분류 |
| 5 그로스 | 퍼널 진단 에이전트 | 퍼널 리포트 + 병목 지목 |
| 6 1인 | 주간 통합 브리핑 에이전트 | 주간 한 장 요약 |

### 1번 퍼포먼스
```
2교시에 만든 광고 대시보드에, 데이터/1_퍼포먼스/campaigns.csv 와 daily.csv 를 읽어서
지난 30일 광고 성과를 분석하는 걸 붙여줘. ROAS 2.0 미만은 경고, 1.0 미만은 위험으로
표시하고, 결과를 대시보드 웹 화면으로 만들어줘. 그 다음 노션에 한 줄 기록하고
디스코드로 요약을 보내줘. 이 전체를 "광고 분석하자" 한 줄로 실행되게 만들어줘.
```
예상 결과: 기준 위반 캠페인 12개(위험 4건, 경고 8건)가 빨강과 주황으로 표시된 대시보드.

### 2번 콘텐츠
```
2교시에 만든 콘텐츠 대시보드에, 데이터/2_콘텐츠/contents.csv 와 keywords.csv 를 읽어서
성과가 좋았던 주제와 검색량 높은 키워드를 뽑고, 그걸 근거로 다음 주 콘텐츠 기획안
5건을 만들어줘. 각 기획안에는 제목 후보 3개와 핵심 메시지를 넣어줘.
결과를 웹 화면으로 만들고 노션에 저장한 다음, 디스코드로 요약을 보내줘.
이 전체를 "콘텐츠 기획하자" 한 줄로 실행되게 만들어줘.
```
예상 결과: 키워드 근거가 붙은 다음 주 기획안 5건.

### 3번 CRM
```
2교시에 만든 고객 대시보드에, 데이터/3_CRM/customers.csv 와 orders.csv 를 읽어서
고객을 등급별로 나누고, 마지막 구매일이 120일 넘은 고객을 이탈 위험군으로 뽑아줘.
등급별 평균 구매금액과 재구매율도 계산해줘. 결과를 웹 화면으로 만들고
노션에 저장한 다음, 디스코드로 요약을 보내줘.
이 전체를 "고객 분석하자" 한 줄로 실행되게 만들어줘.
```
예상 결과: 이탈 위험군 약 200명 추출, 등급 분포는 VIP 29명 · 골드 69명 · 실버 154명 · 일반 248명.

### 4번 브랜드
```
2교시에 만든 브랜드 대시보드에, 데이터/4_브랜드/competitor-snapshot-전일.csv 와
competitor-snapshot-금일.csv 를 비교해서 무엇이 바뀌었는지 찾아줘.
그리고 reviews.csv 를 읽어서 자주 나오는 불만 주제 상위 5개를 뽑아줘.
결과를 웹 화면으로 만들고 노션에 저장한 다음, 디스코드로 요약을 보내줘.
이 전체를 "브랜드 점검하자" 한 줄로 실행되게 만들어줘.
```
예상 결과: A브랜드의 가격 인하와 할인 프로모션 신설, B브랜드의 메인 카피 변경이 변경점으로 잡힌다.

### 5번 그로스
```
2교시에 만든 퍼널 대시보드에, 데이터/5_그로스/funnel.csv 와 sessions.csv 를 읽어서
단계별 전환율과 이탈률을 계산하고, 앞 기간 대비 가장 나빠진 구간을 찾아줘.
결과를 웹 화면으로 만들고 노션에 저장한 다음, 디스코드로 요약을 보내줘.
이 전체를 "퍼널 진단하자" 한 줄로 실행되게 만들어줘.
```
예상 결과: 장바구니에서 결제시작으로 넘어가는 구간이 48%에서 22%로 떨어진 것이 병목으로 잡힌다.

### 6번 1인 마케터
```
2교시에 만든 주간 대시보드에, 데이터/6_1인마케터/weekly.csv 를 읽어서
이번 주 광고비·매출·전환·콘텐츠 성과를 한 장으로 요약해줘.
지난주 대비 늘고 준 것도 표시해줘. 결과를 웹 화면으로 만들고
노션에 저장한 다음, 디스코드로 요약을 보내줘.
이 전체를 "주간 브리핑" 한 줄로 실행되게 만들어줘.
```
예상 결과: 4주 추이가 담긴 한 장 요약.

> **3교시 연결에 성공한 사람** : 위 명령에서 파일 경로 대신 "연결한 계정에서 지난 30일 데이터를 가져와서" 로 바꿔 말하면 된다.
> **연결에 실패한 사람** : 위 명령을 그대로 쓴다. 결과물은 똑같이 나온다.

✅ 체크: "명령 한 줄로 실행했을 때 화면이 나왔나요?"

⏸ 나왔으면 "4단계" 라고 답해 주세요.

## 4단계 · 시나리오 적용 (내 트랙 안에서 택1)

기본 에이전트가 돌면, 내 상황에 맞는 것 하나를 더 붙인다.

| 트랙 | 선택 A | 선택 B |
|:--|:--|:--|
| 1 퍼포먼스 | 매체 3개 합쳐 보기 (예산 재배분 제안) | 주간 성과 리포트 자동 발송 |
| 2 콘텐츠 | SEO 키워드 점수화 (검색의도·난이도) | 경쟁사 콘텐츠와 내 콘텐츠 비교 채점 |
| 3 CRM | 문의 유형 분류하고 답변 초안 만들기 | 세그먼트별 메시지 초안 만들기 |
| 4 브랜드 | 브랜드 톤 기준으로 문구 검수 | 광고 레퍼런스 패턴 정리 |
| 5 그로스 | A/B 실험 3건 판정 | 랜딩 페이지별 성과 비교 |
| 6 1인 | 자주 쓰는 업무를 스킬 파일로 저장 | 다음 주 콘텐츠 주제 뽑기 |

```
선택 A 또는 B 중 하나를 골라 주세요. 둘 다 하실 필요 없습니다.
시간이 빠듯하시면 이 단계를 건너뛰고 5단계로 가셔도 됩니다.
```

> **5번 그로스 트랙 선택 A 참고** : ab-results.csv 에는 실험 3건이 들어 있고 결과가 각각 다르게 나온다.
> AB01은 확실한 승자, AB02는 차이가 작아 판단 보류, AB03은 표본이 부족해 판단 불가다.
> "표본이 부족하면 판단하지 않는다"를 눈으로 보는 것이 이 실습의 핵심이다.

⏸ 붙이셨으면 "5단계" 라고 답해 주세요.

## 5단계 · 점검 (스스로 검사 3가지)

```
🔍 만든 에이전트를 3가지로 점검합니다.
```
