# 5교시 실습 · 후속 액션 + 내 것으로 옮기는 방법 + 발표

> 분석은 끝이 아니다. 숫자를 봤으면 무엇을 끄고, 무엇에 더 넣을지 정해야 한다.
> 오늘 마지막 시간은 '숫자'를 '다음 행동'으로 바꾸고, 이걸 회사에서 내 것으로 만드는 순서를 익힌다.

## 진행 흐름 (3단계)

```
1단계  소개  · 왜 액션까지 가야 하는가 + 안전 게이트          ⏸ 답 대기
2단계  실행  · 액션 붙이기 → 내 것으로 옮기는 방법 익히기      ⏸ 답 대기
3단계  활용  · 발표 + 상호 피드백 + 현업 적용 계획            ⏸ 선택
```

---

## 1단계 · 소개

```
🎬 분석에서 행동까지 연결합니다.

🎯 지금까지 만든 것
  · 2교시 : 대시보드 웹 화면
  · 3교시 : 계정 연결 체험 (또는 실습 파일 그대로)
  · 4교시 : 명령 한 줄로 도는 에이전트

😐 그런데 아직 부족한 것
  리포트를 받아도 "그래서 뭘 하지?" 는 여전히 내 머릿속 숙제로 남는다.
  오늘은 그 숙제까지 에이전트에게 넘긴다.

🔁 이렇게 바뀐다
  · 전 : "ROAS 0.7 캠페인 3개" 라는 숫자만 나온다
  · 후 : "이 3개는 중단 후보. 대신 상위 2개에 예산을 이만큼 옮기자" 까지 나온다

🛡 안전 게이트 (오늘 가장 중요한 원칙)
  에이전트는 '제안까지만' 한다. 실행은 사람이 한다.
  · 예산 변경  → 제안만. 집행은 사람이 승인 후.
  · 광고 집행·중단 → 제안만. 사람이 직접.
  · 고객 메시지 발송 → 초안만. 발송은 사람이 승인 후.
  · 콘텐츠 발행 → 초안만. 발행은 사람이 승인 후.
  제안과 실행 사이에 사람이 한 명 서 있어야 사고가 나지 않는다.
```

⏸ 이해되셨으면 "2단계" 라고 답해 주세요.

## 2단계 · 실행

### 블록 1 · 후속 액션 붙이기 (약 20분)

4교시 에이전트에 "분석 다음에 액션을 제안하라"는 규칙을 얹는다.
본인 트랙의 명령을 그대로 복사해 클로드코드에 넣는다.

**트랙 1 · 퍼포먼스** (중단 후보와 증액안)
```
내 광고 분석 에이전트에 액션 제안을 붙여줘. 기준 미달 캠페인은 중단 후보로 표시하고 근거 숫자를 같이 보여줘. 상위 캠페인에는 예산 증액안을 금액으로 제안하고, 제외할 키워드 후보도 뽑아줘. 실행은 하지 말고 제안만 해.
```

**트랙 2 · 콘텐츠** (제목 리라이트 후보)
```
내 콘텐츠 에이전트에 액션 제안을 붙여줘. 조회 대비 전환이 낮은 콘텐츠를 골라서 제목 대안을 3개씩 만들고, 왜 그 제목이 나은지 한 줄로 설명해줘. 다음 주 기획 주제도 5개 제안해줘. 발행은 하지 말고 초안까지만.
```

**트랙 3 · CRM** (세그먼트 메시지 초안)
```
내 고객 분석 에이전트에 액션 제안을 붙여줘. 이탈 위험 세그먼트별로 보낼 메시지 초안을 만들고, 발송 우선순위를 근거와 함께 정해줘. 실제 발송은 절대 하지 말고 초안만 만들어.
```

**트랙 4 · 브랜드** (대응 메시지 초안)
```
내 브랜드 모니터링 에이전트에 액션 제안을 붙여줘. 감지된 경쟁사 신규 메시지에 대해 우리 대응 메시지 초안을 3개 만들고, 앞으로 추가로 감시할 키워드도 제안해줘. 게시는 하지 말고 초안까지만.
```

**트랙 5 · 그로스** (개선 가설과 A/B 설계안)
```
내 퍼널 진단 에이전트에 액션 제안을 붙여줘. 이탈이 가장 큰 단계에 대해 개선 가설 3개를 만들고, 각 가설을 검증할 A/B 테스트 설계안(무엇을 바꾸고, 무엇을 측정하고, 표본이 얼마나 필요한지)까지 제안해줘.
```

**트랙 6 · 1인 마케터** (이번 주 우선순위 3개)
```
내 주간 브리핑 에이전트에 액션 제안을 붙여줘. 이번 주 성과를 보고 다음 주에 할 일 우선순위 3개를 근거와 함께 정해주고, 각각 몇 시간을 쓸지 시간 배분도 제안해줘.
```

**붙인 다음 확인할 것 3가지**
- 제안에 **근거 숫자**가 같이 붙어 있는가 (숫자 없는 제안은 못 믿는다)
- 제안이 **바로 실행 가능한 문장**인가 ("개선하세요" 말고 "이 캠페인 예산을 이만큼")
- 에이전트가 **실행까지 해버리지 않는가** (제안까지만 멈추는지 확인)

⏸ 액션 제안이 나왔으면 "액션 나옴" 이라고 답해 주세요. 안 나왔으면 어떤 부분이 비었는지 알려 주세요.

### 블록 2 · 회사에 돌아가서 내 것으로 바꾸는 방법 (약 15분)

오늘 만든 에이전트는 실습 파일로 돌고 있다. 이걸 **내 회사 데이터와 내 계정**으로 옮기는 방법을 안내한다.
지금 이 자리에서 다 바꾸지 않는다. 회사 데이터는 대부분 여기 없고, 보안상 가져오면 안 되는 경우도 많다.
**오늘은 순서를 손에 익히고, 실제 전환은 자리에 돌아가서 한다.**

```
📌 왜 지금 안 바꾸나요

  · 회사 데이터를 외부 노트북에 옮기는 것 자체가 문제가 되는 곳이 많습니다.
  · 열 이름과 형식이 회사마다 달라서, 현장에서 맞추다 보면 시간이 다 갑니다.
  · 중요한 건 '내 데이터로 바꾸는 순서'를 아는 것이지, 오늘 바꿔 놓는 게 아닙니다.

  파일을 가져오신 분은 아래 순서를 지금 해보셔도 됩니다. 선택입니다.
```

**① 내 데이터 파일로 바꾸기**

작업 폴더에 내 파일을 넣고 이렇게 말한다.

```
지금 에이전트가 쓰는 실습 데이터를 내가 넣은 파일로 바꿔줘.
열 이름이 다르면 어떤 열을 무엇으로 볼지 먼저 물어보고 진행해줘.
```

```
바뀐 데이터로 처음부터 다시 실행해서 대시보드랑 액션 제안까지 만들어줘.
```

⚠️ **개인정보는 반드시 지우고 쓴다.** 이름, 연락처, 주소, 결제정보 열은 삭제하거나 가린다.
지우기 애매하면 이렇게 시킨다.

```
이 파일에서 이름, 연락처, 주소, 결제정보 열을 지운 사본을 만들어줘. 사본으로만 분석해.
```

**② 내 계정 연결로 바꾸기 (파일 없이 자동으로 받고 싶을 때)**

3교시에서 배운 4단계를 그대로 쓴다. **무엇을 왜 → 키 발급 → 설치 → 작동 확인.**

| 내 상황 | 붙일 것 | 발급 난이도 | 먼저 확인할 것 |
|:--|:--|:--|:--|
| 메타 광고를 운영한다 | Meta Ads | 쉬움 (로그인 동의) | 광고 계정 권한이 분석자 이상인지 |
| 네이버 검색광고를 운영한다 | 네이버 검색광고 | 쉬움 (키 3개 복사) | 광고주 계정으로 로그인했는지 |
| 웹사이트 유입·전환을 본다 | GA4 | 어려움 | 클라우드 프로젝트 생성 권한이 있는지 |
| 구글 검색광고를 운영한다 | 구글애즈 | 어려움 (승인 대기) | 개발자 토큰 신청이 가능한지 |
| 고객·주문이 시트에 있다 | 구글 시트 | 보통 | 시트 열람 권한이 있는지 |
| 경쟁사·리뷰를 모은다 | 웹 수집 | 쉬움 | 무료 키 발급만 하면 됨 |

붙이는 방법을 모르겠으면 클로드코드에 그냥 물어본다.

```
GA4 연결하는 방법을 처음부터 하나씩 알려줘. 나는 개발자가 아니야.
```

**③ 계정을 붙인 뒤 에이전트에 물리기**

이미 만들어 둔 에이전트는 그대로 두고, 데이터가 들어오는 통로만 바꾼다.

```
지금 에이전트가 파일에서 데이터를 읽고 있는데,
방금 연결한 계정에서 지난 30일 데이터를 가져오도록 바꿔줘. 나머지는 그대로 둬.
```

**④ 회사에서 막히는 흔한 경우**

| 상황 | 이렇게 한다 |
|:--|:--|
| 보안상 API 키 발급이 금지돼 있다 | 파일 방식으로 간다. 매체에서 CSV를 내려받아 폴더에 넣으면 끝난다 |
| 계정 권한이 없다 | 권한 있는 담당자에게 조회 권한만 요청한다. 변경 권한은 필요 없다 |
| 열 이름이 실습 파일과 다르다 | "어떤 열이 무엇인지" 알려 주면 바로 붙는다. 파일을 새로 만들 필요 없다 |
| 데이터가 너무 크다 | 최근 한 달치만 잘라서 쓴다. 전체를 넣을 이유가 없다 |
| 혼자 하다 막혔다 | 클로드코드에 에러 메시지를 그대로 붙여넣고 "이거 무슨 뜻이야" 라고 묻는다 |

```
🗓 권장 순서

  1주차   실습 파일 자리에 내 데이터 파일만 바꿔 끼운다. 계정 연결은 아직 안 한다.
  2주차   제일 자주 보는 계정 하나만 연결한다. (메타 또는 네이버 검색광고를 권장)
  3주차   결과를 노션·디스코드로 보내도록 붙인다.
  그 다음  두 번째 에이전트를 만든다.

  한 번에 다 하려다 지치는 것보다, 한 주에 하나씩 붙이는 편이 끝까지 갑니다.
```

⏸ 순서가 이해되셨으면 "이해했어요", 지금 내 파일로 바꿔 보셨으면 "전환 완료" 라고 답해 주세요.

## 3단계 · 활용 (발표 + 현업 적용 계획)

```
🎤 발표 (한 사람 2~3분)
  · 나는 어떤 트랙이고, 무슨 에이전트를 만들었는가
  · 명령 한 줄로 대시보드와 액션 제안이 뜨는 장면 시연
  · 오늘 가장 막혔던 지점과 어떻게 넘었는지

💬 상호 피드백 (듣는 사람이 한 줄씩)
  · "그 제안에 이 숫자를 같이 넣으면 더 설득력 있겠다"
  · "우리 팀이면 여기에 이 단계를 하나 더 붙이겠다"
  · 칭찬보다 '내 업무였다면 이렇게' 가 서로에게 남는다

🗺 현업 적용 계획 (가지고 나갈 것)
  · 오늘 만든 것 : 명령 한 줄로 도는 내 에이전트 1개 + 액션 제안 기능
  · 다음 주부터 무엇을 대체할까 : 지금 손으로 하는 보고서 하나를 정해 이걸로 바꾼다
  · 더 키우기 : 매체·소스 하나 더 연결 / 선택 에이전트 A·B 이어서 제작 / 팀에 공유
  · 재사용 : 오늘 만든 스킬(내 업무를 저장해 둔 파일)을 복사해
    새 매체, 새 고객, 새 프로젝트용으로 계속 찍어낸다
```

**두 번째 에이전트는 무엇으로 할까 (집에 가서 이어서 할 것)**

오늘은 한 개를 만들었다. 다음 것을 고르는 방법까지 손에 쥐어 주고 끝낸다.

```
🧭 다음에 만들 것을 정하는 방법

  마케팅 자동화 매트릭스
  https://likelion-marketing-automation-matrix.netlify.app

  · 오늘 만든 업무 말고, 아직 손으로 하는 반복 업무를 3~5개 더 넣어 보세요.
  · 우선순위가 매겨지고, 각각을 스킬로 만들지 에이전트로 만들지 대시보드로 만들지 나옵니다.
  · 거기서 나온 프롬프트를 그대로 "에이전트 구축하자" 에 붙여넣으면
    오늘과 같은 순서로 두 번째 에이전트를 만들 수 있습니다.

  💡 팀에 공유하실 때도 이 링크가 쓸모 있습니다.
     "뭘 자동화하면 좋을지" 를 말로 설명하는 것보다, 각자 넣어 보게 하는 편이 빠릅니다.
```

⏸ 링크 저장하셨으면 "저장 완료" 라고 답해 주세요.

**마지막 안전 확인 (집에 가기 전에 한 번 더)**

```
· 에이전트는 제안까지만 한다. 예산 변경, 광고 집행, 고객 메시지 발송,
  콘텐츠 발행은 사람이 승인한 뒤 실행한다.
· 계정 연결은 조회 전용을 유지한다.
· 개인정보가 든 원본은 작업 폴더에 두지 않는다.
```

수고하셨습니다. 오늘 여섯 시간으로, 숫자를 보고 끝나던 일이 다음 행동까지 나오는 시스템 한 벌이 되었습니다.
