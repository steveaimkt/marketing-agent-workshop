# 3교시 실습 · 내 계정 연결하기 (MCP 설치)

> MCP = 내 계정과 프로그램을 잇는 '연결선'. 이게 있으면 파일을 직접 안 줘도
> 클로드코드가 계정에서 데이터를 알아서 가져온다. 한 단계씩 확인하며 진행한다.

## 🎯 이 시간의 핵심 (먼저 읽는다)

```
이 시간의 목표는 '연결 성공'이 아니다.
목표는 에이전트 구축 흐름을 끝까지 완주하는 것이다.

· 연결은 도구일 뿐이다. 연결이 안 돼도 4교시, 5교시는 그대로 간다.
· 10분 안에 안 풀리면 손을 든다. 그 자리에서 샘플 데이터로 전환하고 진도를 나간다.
· 구글 계열(Google Ads, GA4)은 API 발급 절차가 많아 개인차가 크다.
  30분에 끝내는 사람도 있고 못 끝내는 사람도 있다. 정상이다. 못 해도 손해가 없다.
· 오늘은 '내 트랙의 우선 연결 1종'만 한다. 나머지는 가이드를 드리니 자습한다.
```

## 진행 흐름 (3단계)

```
1단계  소개  · 연결이 왜 필요한가 + 내 트랙은 무엇을 연결하나   ⏸ 답 대기
2단계  실행  · 우선 연결 1종 설치하고 작동 확인               ⏸ 답 대기
3단계  활용  · 안전 규칙 + 다음 교시 예고                     ⏸ 선택
```

---

## 1단계 · 소개

```
🔌 내 계정을 연결합니다 (MCP 설치).

🎯 왜 필요한가
  2교시 대시보드는 '내가 파일을 넣어야' 돈다. 매주 파일 내려받는 것도 일이다.
  계정을 연결하면, 파일 없이 클로드코드가 계정에서 최신 데이터를 직접 가져온다.

🙅 연결이 필요 없는 경우도 있다
  · 어차피 매달 한 번만 보는 데이터라면, 파일(CSV)로 충분하다.
  · 회사 보안상 API 키 발급이 막혀 있다면, 파일로 간다. 결과물은 똑같이 나온다.

💡 쉬운 말
  '연결(MCP)'은 계정에 '보기 권한'을 주는 것이다.
  열쇠를 새로 만드는 게 아니라, 보기용 출입증을 발급받아 붙이는 느낌이다.

⚠️ 준비 : 해당 계정에 로그인할 수 있어야 하고, 관리자 접근 권한이 있어야 한다.
```

**내 트랙의 우선 연결 (1종만 한다)**

| 트랙 | 우선 연결 | 난이도 | 연결 없이 진행 |
|:--|:--|:--|:--|
| 1 퍼포먼스 | Meta Ads (페이스북·인스타 광고) | 쉬움 | 가능 (광고 CSV) |
| 2 콘텐츠 | 네이버 데이터랩 + 노션 | 쉬움 | 가능 (콘텐츠 CSV) |
| 3 CRM | 구글 시트 + 노션 | 쉬움 | 가능 (고객 CSV) |
| 4 브랜드 | 웹 수집 + 노션 | 쉬움 | 가능 (스냅샷 파일) |
| 5 그로스 | GA4 (구글 애널리틱스) | 중간 | 가능 (트래픽 CSV) |
| 6 1인 | 노션 + 디스코드 | 쉬움 | 가능 |

⏸ 본인 트랙 번호를 "1번" 처럼 답해 주세요.

---

## 2단계 · 실행 (우선 연결 1종 설치)

어떤 연결이든 순서는 똑같다. **무엇을·왜 → 키 발급 → 설치 → 작동 확인.**
이 4단계를 지키면 어떤 도구를 만나도 같은 방식으로 붙일 수 있다.

### 트랙 1 · 퍼포먼스 → Meta Ads

- **무엇을 왜** : 페이스북·인스타 광고 성과(광고비·전환·ROAS)를 계정에서 바로 읽는다.
- **키 발급** : 브라우저에서 로그인·동의만 하면 된다. 별도 키 입력이 없다.
- **설치** : 클로드코드에 이렇게 말한다.
  ```
  메타 광고 계정 연결해줘
  ```
  브라우저가 열리면 직접 클릭해 로그인·동의한다.
- **작동 확인** :
  ```
  내 메타 광고 계정 목록이랑 지난 7일 캠페인 성과 보여줘
  ```

### 트랙 2 · 콘텐츠 → 네이버 데이터랩 + 노션

- **무엇을 왜** : 사람들이 지금 뭘 검색하는지(수요·트렌드)를 가져오고, 기획안을 노션에 쌓는다.
- **키 발급** : 네이버 개발자센터에서 애플리케이션 등록 후 Client ID와 Secret 2개를 받는다. 노션은 로그인 동의만 하면 된다.
- **설치** :
  ```
  네이버 데이터랩이랑 노션 연결해줘
  ```
- **작동 확인** :
  ```
  최근 한 달 '내 브랜드 키워드' 검색 추이 보여주고, 노션 페이지 목록도 보여줘
  ```

### 트랙 3 · CRM → 구글 시트 + 노션

- **무엇을 왜** : 고객·주문 데이터가 들어 있는 시트를 매번 내려받지 않고 바로 읽는다.
- **키 발급** : 구글 로그인 동의 후, 읽을 시트를 지정한다. 노션은 로그인 동의만 한다.
- **설치** :
  ```
  구글 시트랑 노션 연결해줘
  ```
- **작동 확인** :
  ```
  내 고객 시트에서 첫 10행 읽어서 보여줘
  ```

### 트랙 4 · 브랜드 → 웹 수집 + 노션

- **무엇을 왜** : 경쟁사 페이지와 리뷰를 자동으로 긁어오고, 변경점을 노션에 기록한다.
- **키 발급** : 웹 수집 도구는 무료 API 키 1개를 발급받는다. 노션은 로그인 동의만 한다.
- **설치** :
  ```
  웹 수집 도구랑 노션 연결해줘
  ```
- **작동 확인** :
  ```
  경쟁사 홈페이지 주소 하나 줄 테니 지금 화면 내용 요약해줘
  ```

### 트랙 5 · 그로스 → GA4

- **무엇을 왜** : 방문·유입 채널·전환 수를 계정에서 바로 읽어 퍼널을 계산한다.
- **키 발급** : 구글 클라우드에서 프로젝트 생성, API 사용 설정, 서비스 계정 키 발급, GA4 속성에 권한 부여까지 단계가 많다. **여기서 개인차가 가장 크다.**
- **설치** :
  ```
  GA4 연결해줘
  ```
- **작동 확인** :
  ```
  지난 7일 세션 수랑 채널별 유입 보여줘
  ```
- ⏱ **10분 규칙** : 키 발급에서 막히면 바로 멈춘다. `sessions.csv`, `funnel.csv` 샘플로 전환하고 4교시로 간다. 연결은 강의 후 가이드 보고 마저 한다.

### 트랙 6 · 1인 마케터 → 노션 + 디스코드

- **무엇을 왜** : 주간 브리핑을 노션에 기록하고, 요약을 휴대폰(디스코드)으로 받는다.
- **키 발급** : 노션은 로그인 동의. 디스코드는 서버 채널에서 웹훅 주소 1개를 복사한다.
- **설치** :
  ```
  노션이랑 디스코드 연결해줘
  ```
- **작동 확인** :
  ```
  디스코드로 "연결 테스트" 한 줄만 보내줘
  ```

### 공통 · 연결 상태 점검

```
지금 연결된 것들 목록이랑 상태 보여줘
```

안 되는 게 있으면 그 항목은 파일(CSV)로 대체하고 넘어간다.
오늘은 **연결 1종 성공, 또는 샘플 전환 결정**이면 충분하다.

⏸ 연결이 됐으면 "연결됨", 샘플로 가기로 했으면 "샘플로 진행" 이라고 답해 주세요.

## 3단계 · 활용

```
✅ 이제 클로드코드가 내 계정을 직접 볼 수 있다 (보기 전용).

🛡 안전 규칙 (꼭 지킨다)
  · 계정은 '조회'만 한다. 예산 변경, 광고 집행, 메시지 발송, 콘텐츠 발행,
    삭제, 결제는 자동화하지 않는다.
  · 조치는 사람이 직접 하거나, 내가 명확히 시켰을 때만 한다.
  · 발급받은 키는 남에게 공유하지 않는다. 화면 공유 중에는 키 화면을 띄우지 않는다.
  · 고객 이름, 연락처, 주소, 결제정보가 담긴 자료는 지우고 다룬다.

📌 오늘 못 한 연결
  · 가이드를 드린다. 회사에 돌아가 같은 4단계(무엇을 왜 → 키 발급 → 설치 → 확인)로 붙이면 된다.
  · 연결이 없어도 4교시 에이전트는 파일 기준으로 똑같이 완성된다.

▶ 다음 : 4교시에서 '2교시 대시보드 + 3교시 연결 + 내 규칙'을 하나로 합쳐,
  명령 한 줄이면 분석부터 리포트까지 실행하는 에이전트로 완성한다.
```
