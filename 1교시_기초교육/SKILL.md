# 1교시 기초 교육 손풀기 실습 (초보자용 · 6트랙 공용)

> 게이트형 가이드다. 한 번에 다 실행하지 말고 STEP마다 수강생 확인 후 다음으로.
> 3블록: 세팅(공통) → 트랙 선택 + 문서(CLAUDE.md) → 트랙별 시각화 실습(골라서).
> 목표: ① 클로드가 내 폴더의 내 직군 데이터를 읽게 하고 ② 숫자를 눈에 보이는 결과물로 바꾼다.
> 오후 3단계(대시보드 → 연결 → 에이전트)의 근육을 미리 푸는 시간.

## 진행 원칙 (클로드가 지킬 것)

1. 한 STEP씩 안내하고 수강생의 완료 응답을 기다린다.
2. 복붙 우선: 입력할 문장은 코드 블록으로 준다.
3. 예상 결과 먼저: 실행 전 "이런 화면이 나오면 성공"을 알려주고, 조금 달라도 형태가 같으면 성공이라고 안심시킨다.
4. 어려운 말은 그 자리에서 쉬운 말로 풀어 준다.
5. 트랙이 정해지기 전(STEP 2-0 이전)에는 트랙별 내용을 미리 보여주지 않는다.

## 시작 멘트 (스킬 호출 시 가장 먼저 출력)

```
안녕하세요. 1교시 기초 교육 손풀기입니다. 3블록, 약 50~60분입니다.

  블록 1  기본 세팅        로그인 · 한글팩 · 권한 모달 (전원 동일)      ~15분
  블록 2  내 트랙 데이터    트랙 선택 · 작업 폴더 · CLAUDE.md · 구조 파악  ~20분
  블록 3  시각화 실습(골라서) 내 직군 데이터를 표 · 차트 · 색으로          나머지

오늘은 직군 6개 트랙(1 퍼포먼스 · 2 콘텐츠 · 3 CRM · 4 브랜드 · 5 그로스 · 6 1인마케터)으로
나뉘어 진행합니다. 트랙은 블록 2에서 고릅니다. 만드는 방법은 전원 같고, 데이터와 결과물만 다릅니다.

약속 2가지
  · 터미널·코드는 안 씁니다. 전부 이 채팅창에서 한국어로.
  · 결과는 매번 조금 다릅니다. 형태가 같으면 성공입니다.

준비되셨으면 "시작"이라고 입력해 주세요.
```

# 블록 1. 안티그래비티 기본 세팅 (~15분 · 전 트랙 공통)

목표: 도구를 켜고, 한글로 바꾸고, 파일을 만들 수 있는 상태까지.

## STEP 1-0. 화면 3구역 안내
```
화면을 세 곳으로 봅니다.
  · 왼쪽    파일 탐색기   우리가 만든 폴더·파일이 쌓이는 곳
  · 가운데  편집기        파일을 클릭하면 내용이 열리는 곳
  · 오른쪽  채팅창        클로드에게 말 거는 곳. 오늘 지시는 전부 여기서.
```
포인트: "터미널·코드는 안 씁니다. 채팅창에 한국어로 부탁만 하면 됩니다."
✅ 체크: "채팅창이 어디인지 찾으셨나요?"

## STEP 1-1. 로그인 확인
따라하기:
```
안녕, 지금 로그인 잘 돼 있어?
```
포인트: "무료 플랜은 안 됩니다. claude.ai Pro 이상 계정으로 로그인돼 있어야 끝까지 진행됩니다."
✅ 체크: "인사에 답이 돌아왔나요?"

## STEP 1-2. 한글 언어팩 (IDE 한글화)
경로: 왼쪽 확장(Extensions) 아이콘 → `Korean` 검색 → "Korean Language Pack" 설치 → "Change and Restart".
안 보이면 채팅창에: `이 IDE를 한글로 바꾸고 싶어. Korean Language Pack 설치법 알려줘`
포인트: "화면만 편해지는 겁니다. 안 되면 건너뛰어도 실습엔 지장 없습니다."
✅ 체크: "메뉴가 한글로 보이나요?"

## STEP 1-3. 파일 생성 + 권한 모달 체험
따라하기:
```
hello.txt 만들고 '첫 마케팅 자동화 테스트 성공'이라고 써줘
```
예상: 권한 모달이 뜨면 "허용" 클릭 → 왼쪽에 hello.txt 생성.
포인트: "방금 뜬 권한 모달은 안전장치입니다. 오후에 실제 계정을 다룰 때 결정적 역할을 합니다."
✅ 체크: "왼쪽 탐색기에 hello.txt가 보이나요?"

# 블록 2. 트랙 선택 + 내 데이터 준비 + CLAUDE.md (~20분)

목표: 내 직군 트랙을 고르고, 작업 폴더에 내 트랙 데이터를 넣고, 지침서(CLAUDE.md)를 만든 뒤, 데이터 구조를 파악한다.

## STEP 2-0. 내 트랙 고르기 (여기서 길이 갈립니다)

아래 표를 그대로 출력한다.
```
어떤 마케터로 오늘을 보내실까요? 번호 하나만 골라 주세요.

  1  퍼포먼스   광고 성과를 분석한다             campaigns.csv · daily.csv · creatives.csv
  2  콘텐츠     콘텐츠와 키워드 성과를 본다        contents.csv · keywords.csv · competitor-posts.csv
  3  CRM        고객을 세그먼트로 나눈다           customers.csv · orders.csv · cs-tickets.csv
  4  브랜드     경쟁사와 고객의 소리를 모은다       reviews.csv · competitor-snapshots/ · brand-guide.md
  5  그로스     퍼널 어디서 새는지 찾는다          sessions.csv · funnel.csv · landing.csv · ab-results.csv
  6  1인마케터  광고·콘텐츠·매출을 한 번에 본다      weekly.csv · tasks.md

고민되면 "지금 회사에서 매주 열어 보는 숫자"가 무엇인지로 고르세요.
```
⏸ 번호 하나만 답해 주세요. (예: "3")

클로드 실행 규칙: 답을 받으면 그 트랙 번호를 기억하고, 이후 모든 안내에서 그 트랙 데이터와 메뉴만 쓴다. 2교시에도 같은 트랙으로 이어간다.

## STEP 2-1. 작업 폴더 생성
```
"내-마케팅분석" 폴더를 만들어줘. 오늘 실습은 여기서 할 거야
```
✅ 체크: "왼쪽 탐색기에 폴더가 생겼나요?"

## STEP 2-2. 내 트랙 데이터 넣기 (두 갈래)

- 본인 데이터가 있으면: 내보내기 파일(엑셀/CSV)을 `내-마케팅분석` 폴더로 드래그.
- 없으면 채팅창에 아래를 붙여 넣는다. `{번호}_{트랙명}` 부분은 STEP 2-0에서 고른 것으로.

```
실습파일/데이터/{번호}_{트랙명} 폴더의 파일을 전부 내-마케팅분석 폴더로 복사해줘
```

폴더 이름은 STEP 2-0 표와 같다. 예: 트랙 3이면 `실습파일/데이터/3_CRM`. 헷갈리면 "내 트랙 폴더 찾아서 복사해줘"라고만 해도 된다.
포인트: "샘플은 전부 실습용 가상 데이터입니다. CRM 트랙 고객 데이터도 익명화된 가짜 데이터입니다."
✅ 체크: "폴더에 내 트랙 파일이 보이나요?"

## STEP 2-3. CLAUDE.md 생성 (네 블록)

포인트: "지침서는 손으로 안 씁니다. 말로 시키면 초안을 만들고, 우리는 네 블록만 확인합니다. 이 네 블록이 오후 에이전트가 지킬 규칙의 뿌리입니다."
따라하기 (`{내 직군}` 자리에 본인 트랙 이름을 넣는다):
```
내-마케팅분석 폴더에 CLAUDE.md를 만들어줘.
나는 {내 직군} 마케터고, 이 폴더에서 내 데이터를 분석할 거야.
답변은 한국어로, 금액은 원 단위로, 고객·계정 정보는 익명으로 다뤄줘.
```

클로드 실행 규칙: 네 블록으로 생성 = `## 나는 누구` · `## 이 폴더에서 할 일` · `## 답변 규칙` · `## 우리만의 기준`. "우리만의 기준"은 블록3 실습 F에서 직접 채운다고 예고한다. 트랙 3(CRM)이면 "개인정보는 이름·연락처·주소 없이 다룬다" 한 줄을 답변 규칙에 추가한다.
✅ 체크: "폴더에 CLAUDE.md가 생기고 네 블록이 보이나요?"

## STEP 2-4. 데이터 구조 파악 (컬럼 분석표)
따라하기:
```
이 폴더의 데이터를 읽고, 어떤 칸이 있고 무슨 뜻인지 표로 정리해줘
```
클로드 실행 규칙: 파일명 / 행·열 수 / 기간 / 컬럼 분류(구분·수치·비율) / 품질 메모.
✅ 체크: "정리된 칸이 실제 데이터와 맞나요? 이 표는 2교시 대시보드 설계에 그대로 씁니다."

# 블록 3. 시각화 실습 · 골라서 (남은 시간)

> 순서대로 다 하지 않는다. 6개 중 시간 되는 만큼 고른다. ★는 추천(2교시와 직결).
> 핵심: "숫자 → 눈에 보이는 결과물". 표·차트·색으로 바꿔 회의에서 바로 쓰게.
> 클로드는 **고른 트랙의 메뉴만** 출력한다. 다른 트랙 메뉴는 보여주지 않는다.

공통 안내 문구를 먼저 출력:
```
블록 3 · 시각화 실습 메뉴입니다. 순서대로 다 하지 않아도 됩니다.
★ 두 개만 해도 2교시로 넘어가는 데 충분합니다.
하나 고르면 문장을 그대로 복사해 채팅창에 붙여 넣으세요.
```

## 트랙 1. 퍼포먼스 메뉴

```
A ★ 캠페인 성과 한눈에 (표)
campaigns.csv에서 ROAS 상위 10개 캠페인을 HTML 표 한 장으로 만들어줘. 순위·캠페인·광고비·매출·ROAS 열로. 만들고 브라우저로 열어줘
B ★ ROAS 막대 차트
campaigns.csv에서 ROAS 상위 10개를 막대그래프 HTML로 보여줘. 숫자도 막대 옆에, ROAS 2.0에 기준선도 그어줘. 만들고 브라우저로 열어줘
C 고효율/저효율 분류
campaigns.csv 캠페인을 ROAS 기준으로 고효율과 저효율로 나누고 핵심을 3줄로 요약해줘. 목적이 도달·트래픽인 캠페인은 수익률만으로 판단하지 말라고 주석 달아줘
D 요일·채널 히트맵
daily.csv의 채널 x 요일로 ROAS를 색으로 표현한 히트맵 HTML을 만들어줘. 진할수록 성과 좋은 걸로. 만들고 브라우저로 열어줘
E 소재 TOP·BOTTOM
creatives.csv에서 성과 상위 5개와 하위 5개 소재를 카드 형태 HTML로 비교해줘. 무엇이 갈랐는지 한 줄씩 붙여줘
F 우리만의 기준 효과  → 아래 'F 공통' 순서를 따른다
```

## 트랙 2. 콘텐츠 메뉴

```
A ★ 전환율 상위 콘텐츠 (표)
contents.csv에서 조회 대비 전환율 상위 10개 콘텐츠를 HTML 표로 만들어줘. 순위·제목·채널·조회·체류·전환율 열로. 만들고 브라우저로 열어줘
B ★ 채널별 효율 비교
contents.csv를 채널별로 묶어 평균 전환율을 막대그래프 HTML로 보여줘. 숫자도 막대 옆에. 만들고 브라우저로 열어줘
C 키워드 급상승 라인 차트
keywords.csv에서 최근 검색량이 가장 많이 오른 키워드 10개를 라인 차트 HTML로 보여줘. 급상승 키워드는 색을 다르게 해줘
D 콘텐츠 유형 분류
contents.csv의 콘텐츠를 성과 기준으로 잘 먹힌 유형과 아닌 유형으로 나누고, 공통점을 각각 3줄로 요약해줘
E 경쟁사 게시물 비교
competitor-posts.csv와 contents.csv를 비교해서 우리 콘텐츠가 인게이지먼트에서 밀리는 지점을 표로 정리해줘
F 우리만의 기준 효과  → 아래 'F 공통' 순서를 따른다
```

## 트랙 3. CRM 메뉴

```
A ★ 재구매율 계산 (요약 표)
customers.csv와 orders.csv를 합쳐서 재구매율과 평균 구매주기를 계산하고 요약 표 HTML로 만들어줘. 고객 이름은 표시하지 말아줘
B ★ 세그먼트 분포
customers.csv 고객을 구매 금액대별로 나눠 분포를 막대그래프 HTML로 보여줘. 각 구간 인원수도 같이. 만들고 브라우저로 열어줘
C 이탈 위험 리스트
orders.csv에서 마지막 구매 이후 경과일이 긴 순으로 이탈 위험 고객 20명을 표로 뽑아줘. 고객 ID만 쓰고 개인정보는 빼줘
D 구매 금액대 히트맵
가입 시기 x 구매 금액대로 고객 수를 색으로 표현한 히트맵 HTML을 만들어줘. 진할수록 인원이 많은 걸로
E CS 문의 주제 분류
cs-tickets.csv 문의를 주제별로 분류하고 자주 나오는 상위 5개 주제를 카드 HTML로 정리해줘
F 우리만의 기준 효과  → 아래 'F 공통' 순서를 따른다
```

## 트랙 4. 브랜드 메뉴
